# Лабораторная работа 3: Поиск пути в лабиринте

## Описание проекта

Данный проект реализует алгоритм поиска пути в лабиринте для лабораторной работы по курсу программирования на C++. Проект разработан в рамках **Кейса 5 "Поиск пути в лабиринте"** и демонстрирует безопасную работу с динамическими структурами данных.

## Цели проекта

### Образовательные цели
- Освоить методику разработки надежного и эффективного кода для работы с динамическими матрицами
- Сформировать практические навыки работы с динамической памятью в C++
- Развить навыки командной работы и код-ревью

### Технические цели
- Реализовать корректное управление памятью (выделение/освобождение)
- Разработать алгоритм поиска пути в лабиринте с использованием BFS
- Обеспечить обработку граничных условий и ошибочных ситуаций
- Интегрировать инструменты проверки памяти (Valgrind, AddressSanitizer)

## Функциональность

### Основные функции

1. **`create_matrix(rows, cols)`** - создание динамической матрицы
2. **`free_matrix(&matrix)`** - безопасное освобождение памяти
3. **`create_sample_maze()`** - создание тестового лабиринта 5x5
4. **`print_maze(maze)`** - визуализация лабиринта в консоли
5. **`is_reachable(maze, start, end)`** - проверка существования пути

### Алгоритм поиска пути

Реализован **поиск в ширину (BFS)** с следующими особенностями:
- Отслеживание посещенных клеток для избежания циклов
- Проверка граничных условий
- Обработка невалидных входных данных
- Оптимизированное использование памяти

## Примеры использования

### Базовый пример
```cpp
#include "maze_solver.h"

int main() {
    // Создание тестового лабиринта
    Matrix maze = create_sample_maze();

    // Визуализация
    print_maze(maze);

    // Проверка пути
    bool path_exists = is_reachable(maze, 1, 1, 3, 3);

    // Освобождение памяти
    free_matrix(&maze);

    return 0;
}
```

## Проблемы и решения

### Распространенные проблемы
1. **Утечки памяти** - убедитесь, что каждый `create_matrix` имеет парный `free_matrix`
2. **Выход за границы** - проверяйте индексы перед доступом к элементам
3. **Бесконечная рекурсия** - используйте BFS вместо DFS для больших лабиринтов
